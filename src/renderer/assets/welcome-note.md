# Welcome

This note is the default starting point for welcome info, release updates, and small product messages.

BridgeGit keeps the parts of your daily repo workflow close together:

- inspect changes and stage files
- review diffs without leaving the workspace
- run shell commands in parallel tabs
- keep short notes, prompts, and scratch markdown nearby

## Quick Start

1. Open a repository from the left panel.
2. Review changed files and stage what you want.
3. Use shell tabs for Git, build, or tooling commands.
4. Keep notes in extra markdown tabs when you need context next to the repo.

## Current Build Focus

- offline desktop workflow
- fast Git + terminal loop
- markdown notes with live preview
- local session persistence

## Product Direction

BridgeGit is being shaped as a focused desktop workspace first.
The free core stays centered on local Git, shell, and notes.

## Tips

- Double-click a tab title to rename it.
- Create a new notes tab when you want scratch space next to the terminal.
- Use preview mode when you want to read notes without editor chrome.

Open a repo and start with the shell tab when you're ready.
